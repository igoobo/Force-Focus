# 신규 기능 제안: 능동적 번아웃 방지 및 몰입 흐름 보호 (Positive Intervention System)

> **상태**: Proposed (제안됨)
> **적용 대상**: Desktop Agent (FSM Layer, `core/state.rs`, `core/app.rs`)
> **예상 작업 시간**: Medium

---

## 1. 개요 (Overview)

현재 데스크톱 에이전트의 FSM 루프(`FOCUS` ↔ `DRIFT` ↔ `DISTRACTED`)는 사용자가 '업무 외 행위'를 할 때 이를 감지하고 차단하는 **부정적 개입(Negative Intervention)**에 초점이 맞춰져 있습니다. 
하지만 고수준의 생산성 관리 기능은 사용자가 번아웃에 빠지지 않도록 장기 몰입을 보호하는 것입니다. 본 제안은 사용자가 장시간 집중 상태(`FOCUS`)를 유지했을 때, 데스크톱 에이전트가 흐름이 끊기지 않는 최적의 타이밍을 찾아 선제적으로 "휴식(Refresh)"을 권고하는 **긍정적 개입**을 추가하는 것입니다.

## 2. 해결하고자 하는 문제점

- **번아웃 위험성 방치**: 사용자가 3~4시간 연속으로 모니터만 보고 있어도, '딴짓'만 하지 않으면 현재 시스템은 아무런 개입을 하지 않습니다.
- **Intervention Fatigue**: 지속적인 경고성 알림은 사용자에게 피로감을 줄 수 있습니다. 반면 휴식을 권장하는 시스템은 에이전트에 대한 사용자 친화도를 높입니다.

## 3. 핵심 기능 동작

1. **상태 관리**: 연속적인 `FOCUS` 세션의 누적 시간을 트래킹합니다 (예: 50분).
2. **타이밍 포착**: 단순 시간이 만료되었다고 오버레이를 띄우면 오히려 몰입을 방해할 수 있습니다. AI(ML 피처)가 다음의 힌트를 이용해 "휴식하기 가장 좋은 타이밍"을 찾아냅니다.
   - 키보드/마우스 입력이 일시적으로 멈추는 구간 (`silence_sec` 스파이크)
   - 앱/문서 컨텍스트 스위칭이 일어날 때 (예: 코드 에디터를 닫고 브라우저를 켤 때)
3. **상태 전이**: FSM에 `FATIGUE` 또는 `REST_RECOMMENDED` 상태를 추가하여, 거부감이 없는 부드러운 UI 형태(예: 화면 테두리가 부드러운 그린 톤으로 빛남, 토스트 메시지 등)로 휴식을 유도합니다.

## 4. 구조적 변경 사항 (Architecture Impact)

- `core/app.rs`: 루프 내에서 연속된 Focus 시간을 산출하는 변수(`continuous_focus_min`) 추가 필요.
- `core/state.rs`: 기존 FSM에 신규 상태 추가 및 `Gauge` 계산 로직을 휴식 타이머와 연동.
- `frontend/`: 기존의 붉은 계열의 Distracted 오버레이가 아닌, 휴식과 안정을 주는 테마 디자인(플로팅 혹은 은은한 오버레이 효과) 컴포넌트 추가.
